# `research/` — what's in this directory

This folder mixes three things and the distinction matters: **generic primitives** that any
user runs with their own parameters, **reference drivers** that pin the specific sweeps we
ran during fsvlm's development, and **snapshot data** (the append-only ground truth + the
pre-registered taxonomy + the hypothesis queue). Read the table below before running anything.

## Generic primitives — start here

These take parameters. Use them on your own data.

| File | What it does | Parameters |
|---|---|---|
| [`run_sweep.sh`](run_sweep.sh) | Generic sweep wrapper. Composes `fsvlm train` + `fsvlm validate` per cell, appends provenance-stamped rows to the result log. **The entry point most users want.** | `--datasets`, `--categories`, `--n-values`, `--seeds`, `--label-source`, `--recipe-version`, `--output` |
| [`dataset_size_sweep.py`](dataset_size_sweep.py) | The actual sweep engine that `run_sweep.sh` wraps (707 lines). Generic — works with any registered backend / `LabelReader`. | same as above; see `--help` |
| [`verdict.py`](verdict.py) | Pure computation. Reads result rows, classifies each cell as `keep` / `discard` / `noop` / `new_baseline` against the prior baseline. No git / no model / no I/O on the working tree. | `--results`, `--recipe-version`, `--write` |
| [`plots.py`](plots.py) | Generic figure generator. Reads result rows, emits per-category AUROC-vs-N curves and CSV/Markdown/LaTeX summary tables. | `--results`, `--output`, `--figures`, `--recipe`, `--format` |
| [`tiered_validation.py`](tiered_validation.py) | Generic zero-shot → few-shot → full-train comparison on the same fixed test split. The proof-of-lift protocol. | `--dataset`, `--category`, `--backend`, `--few-shot-n`, `--seeds`, `--results-path` |
| [`icl_baseline.py`](icl_baseline.py) | Generic in-context-learning baseline. Runs the base VLM with reference images in the context window (no fine-tuning). Pairs 1:1 with sweep rows for FT-vs-ICL ablations. | `--categories`, `--n-shots`, `--seeds`, `--output` |
| [`agent_labeler.py`](agent_labeler.py) | Generic per-image natural-language description generator. Useful for the `agent` label source in `run_sweep.sh`. | `<dataset> <category>` |
| [`datasets/download_*.sh`](datasets/) | One per dataset. New datasets get a new `download_<name>.sh`. | (none) |

## Reference drivers — our specific research arc

These pin the **exact sweeps we ran** to produce the rows in `dataset_size_results.json`.
They're hand-written shell scripts that hardcode the categories, recipe knobs, and N-values
for our specific hypotheses. **You don't need to run any of these to use fsvlm — they're here
so anyone can reproduce our published numbers and so you have concrete examples of how to
compose the generic primitives.**

| File | The pass it ran |
|---|---|
| [`passes/pass2_driver.sh`](passes/pass2_driver.sh) | Label-source ablation: thin / metadata / agent labels at N=30, 3 categories, 2 seeds |
| [`passes/pass2b_driver.sh`](passes/pass2b_driver.sh) | Recipe bump test (rank=16, LR=1e-4, alpha=8) — null result |
| [`passes/pass3_driver.sh`](passes/pass3_driver.sh) | The headline sweep: AUROC-vs-N curves on hazelnut/candle/pcb at N ∈ {2, 3, 5, 10, 20, 30, 40, 60, 100} × 3 seeds |
| [`passes/pass3_resume.sh`](passes/pass3_resume.sh) | Resume helper after a power disruption mid-sweep |
| [`passes/pass4_driver.sh`](passes/pass4_driver.sh) | Tier A completion: 21 remaining MVTec + VisA categories at N=30 (planned for v0.2 coverage) |
| [`passes/pass5b_driver.sh`](passes/pass5b_driver.sh) | ICL-vs-FT ablation at N=2: same model, same splits, fine-tune vs in-context |
| [`passes/pass6_driver.sh`](passes/pass6_driver.sh) | Stratified-sampling rescue attempt for metal_nut (null result) |

To run any of them: `bash research/passes/<name>.sh` from the repo root. They each `cd` to
the repo root via `git rev-parse --show-toplevel` so the location doesn't matter.

To write your own driver for your own hypothesis: copy any of the above as a starting point
and edit the `--datasets` / `--categories` / `--n-values` / `--seeds` / `--recipe-version`
arguments. The driver's job is to encode "here is the specific question I'm answering and
the cohort tag I'm grouping the rows under."

## Snapshot data — read-only ground truth

These are committed evidence, not generated at runtime.

| File | What it carries |
|---|---|
| [`dataset_size_results.json`](dataset_size_results.json) | **159 result rows**, append-only. Every row has `auroc`, `f1`, `precision`, `recall`, `accuracy`, `threshold`, `num_test`; v0.1+ rows also have `git_hash`, `git_short`, `recipe_version`, `status`. |
| [`tiered_results.json`](tiered_results.json) | Pre-v0.1 zero-shot / few-shot / full-train comparison data. |
| [`agent_labels.json`](agent_labels.json) | **5,450 cached per-image descriptions** generated by `agent_labeler.py` on MVTec + VisA. ~2.2 MB. |
| [`defect_taxonomy.json`](defect_taxonomy.json) | Pre-registered `distinctive` / `subtle` / `mixed` tagging of every defect subtype. **Frozen 2026-04-20T12:30** — editing it post-hoc invalidates the rows it explains. |
| [`queue.json`](queue.json) | The hypothesis queue (one entry per pass) with `status`, `verdict_summary`, and the question each pass tested. |
| [`phase0_baseline.json`](phase0_baseline.json) | Phase 0 ML-validation-gate baseline data (AUROC ≥ 0.90 on MVTec hazelnut). |
| [`phase0_validation.py`](phase0_validation.py) | The Phase 0 validation script (1092 lines). Project-specific — kept as a reference example of what an end-to-end validation gate looks like. |
| [`requirements-phase0.txt`](requirements-phase0.txt) | Phase 0 dependency pin file (kept for the historical record). |

## Verifying the data is what we say it is

If you want to confirm every headline number in the README is reproducible from the rows
above, run:

```bash
python scripts/verify_readme_numbers.py
```

That script re-derives every percentage and AUROC value from `dataset_size_results.json`
with the explicit recipe-version filter shown next to each number. If your output doesn't
match the README, file an issue.

## Reproducing our exact sweeps end-to-end

```bash
# 1. Datasets (MVTec AD requires email registration; place under research/mvtec_data/)
bash research/datasets/download_visa.sh
bash research/datasets/download_deeppcb.sh

# 2. Run the headline AUROC-vs-N sweep (the one that produced the README's hero figure)
bash research/passes/pass3_driver.sh

# 3. Classify the new rows
python research/verdict.py --results research/dataset_size_results.json --write

# 4. Regenerate figures + tables
python research/plots.py --results research/dataset_size_results.json --output docs/figures/
```

Or run any other `passes/passN_driver.sh` instead of step 2 to reproduce a different sweep.
